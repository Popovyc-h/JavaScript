## Завдання 1. Менеджер налаштувань теми (theme config)

Фронтенд-розробники часто працюють з темами — об'єктами, що описують кольори, відступи, шрифти.

```javascript
// Базові налаштування теми
const defaultTheme = {
    primaryColor: '#007bff',
    secondaryColor: '#6c757d',
    fontSize: 16,
    borderRadius: 4,
}
```

**Що потрібно зробити:**

1. Створити функцію `createTheme(overrides)`, яка приймає об'єкт `overrides` і повертає новий об'єкт теми, де:
    - Властивості з `overrides` перезаписують однойменні властивості з `defaultTheme`.
    - Властивості, яких немає в `defaultTheme`, ігноруються (щоб випадково не додати щось зайве).
    - Вихідний `defaultTheme` не змінюється.

2. Створити функцію `getCssVars(theme)`, яка приймає об'єкт теми і повертає **новий** об'єкт, де ключі перетворені на CSS-змінні: `primaryColor` → `"--primary-color"`, `fontSize` → `"--font-size"` тощо. Значення залишаються тими ж.

3. Створити функцію `updateTheme(theme, path, value)`, яка змінює вкладену властивість за шляхом, переданим через крапку. Наприклад:

```javascript
const theme = {
    colors: {
        primary: '#007bff',
        secondary: '#6c757d',
    },
    spacing: { unit: 8 },
}

updateTheme(theme, 'colors.primary', '#ff0000')
// theme.colors.primary тепер "#ff0000"
```

> **Підказка:** шлях на кшталт `"colors.primary"` треба розбити через `split(".")` і пройтися по рівнях вкладеності.

## Завдання 2. Злиття профілів користувачів (user merge)

У реальному проєкті дані про користувача часто надходять з різних джерел: реєстраційна форма, соціальна мережа, CRM тощо.

```javascript
const profileFromForm = {
    name: 'Іван',
    email: 'ivan@example.com',
    age: 28,
}

const profileFromSocial = {
    name: 'Ivan Petrenko',
    avatar: 'https://example.com/avatar.jpg',
    socialId: '12345',
}

const profileFromCRM = {
    email: 'ivan.petrenko@work.com',
    role: 'admin',
    preferences: {
        language: 'uk',
        notifications: true,
    },
}
```

**Що потрібно зробити:**

1. Створити функцію `mergeProfiles(...profiles)`, яка приймає будь-яку кількість об'єктів профілів, і повертає один зведений профіль. Якщо ключі повторюються, пріоритет має профіль, який іде **пізніше** в аргументах.

2. Створити функцію `deepFreeze(obj)`, яка робить об'єкт **та всі його вкладені об'єкти** повністю замороженими (`Object.freeze`). Це потрібно, щоб випадково не змінити профіль після злиття.

> **Підказка:** обійдіть усі властивості циклом `for..in`. Якщо значення властивості — об'єкт (але не `null`), викличте `deepFreeze` для нього рекурсивно.

3. Створити функцію `diffProfiles(original, updated)`, яка порівнює два профілі й повертає об'єкт, де ключі — це назви змінених властивостей, а значення — об'єкт `{ from, to }`. Вкладені властивості теж треба порівнювати (рекурсивно). Якщо змін немає — повернути `null`.

**Приклад:**

```javascript
diffProfiles({ name: 'Іван', age: 28, prefs: { lang: 'uk' } }, { name: 'Іван', age: 30, prefs: { lang: 'en' } })
// { age: { from: 28, to: 30 }, prefs: { lang: { from: "uk", to: "en" } } }
```

## Завдання 3. Система плагінів (plugin config)

У збірниках модулів (Webpack, Vite) та редакторах (VS Code) плагіни передають свій конфіг, який треба вміло об'єднувати з дефолтним.

```javascript
const defaultPluginConfig = {
    output: {
        filename: 'bundle.js',
        path: '/dist',
        sourceMap: false,
    },
    rules: {
        js: { enabled: true, loader: 'babel' },
        css: { enabled: false, loader: 'css' },
    },
    plugins: [],
    mode: 'development',
}
```

**Що потрібно зробити:**

1. Створити функцію `deepMerge(target, source)`, яка робить **глибоке злиття** двох об'єктів:
    - Якщо властивість є об'єктом в обох — зливати рекурсивно.
    - Якщо властивість є примітивом в одному з них — значення з `source` перезаписує.
    - Вихідні об'єкти не змінюються — повертається новий об'єкт.

2. Створити функцію `hasCircularReference(obj)`, яка перевіряє, чи є в об'єкті циклічні посилання (наприклад, `obj.self = obj`). Повертає `true` або `false`.

> **Підказка:** використовуйте додатковий об'єкт `seen` (або `Set` сторонніх бібліотек, але ми поки що не знаємо Set — можна звичайний об'єкт), щоб запам'ятовувати вже відвідані об'єкти під час рекурсивного обходу.

3. Створити функцію `pick(obj, keys)`, яка приймає об'єкт і масив рядків-ключів, і повертає **новий** об'єкт, що містить тільки ті властивості, чиї ключі є в `keys`. Це корисно, коли з API приходить великий об'єкт, а нам треба взяти лише кілька полів.

```javascript
pick({ a: 1, b: 2, c: 3, d: 4 }, ['a', 'c'])
// { a: 1, c: 3 }
```

## Завдання 4. Система локалізації (i18n)

У багатомовних вебдодатках переклади зберігаються у вигляді вкладених об'єктів.

```javascript
const translations = {
    uk: {
        greeting: 'Привіт',
        farewell: 'До побачення',
        menu: {
            home: 'Головна',
            about: 'Про нас',
            contact: 'Контакти',
        },
    },
    en: {
        greeting: 'Hello',
        farewell: 'Goodbye',
        menu: {
            home: 'Home',
            about: 'About',
            contact: 'Contact',
        },
    },
}
```

**Що потрібно зробити:**

1. Створити функцію `t(lang, path)`, яка приймає мову (напр. `"uk"`) і шлях через крапку (напр. `"menu.home"`), і повертає відповідний переклад. Якщо переклад не знайдено — повертає рядок `"🔍 [missing translation]"`.

```javascript
t('uk', 'menu.home') // "Головна"
t('fr', 'greeting') // "🔍 [missing translation]"
t('uk', 'menu.nonexist') // "🔍 [missing translation]"
```

2. Створити функцію `addTranslation(lang, path, value)`, яка додає новий переклад до існуючого об'єкта `translations`. Шлях `path` може бути вкладеним (напр. `"menu.products.title"`).

```javascript
addTranslation('uk', 'menu.products.title', 'Товари')
// translations.uk.menu.products.title тепер "Товари"
```

3. Створити функцію `flattenTranslations(lang)`, яка приймає мову, і повертає **плоский** об'єкт, де ключі — це шляхи через крапку, а значення — переклади.

```javascript
flattenTranslations('uk')
// {
//   "greeting": "Привіт",
//   "farewell": "До побачення",
//   "menu.home": "Головна",
//   "menu.about": "Про нас",
//   "menu.contact": "Контакти",
// }
```

## Додаткові вимоги

1. Усі функції мають бути **чистими** (не модифікувати вхідні дані), крім `updateTheme` та `addTranslation` (вони свідомо змінюють об'єкт).
2. Не використовувати методи масивів (`map`, `reduce`, `filter`, `forEach`) — тільки цикли `for`, `for..in`.
3. Коментувати код не потрібно, але назви змінних мають бути змістовними.
4. Тестувати кожну функцію хоча б на 2-3 прикладах (можна просто викликати в кінці файлу і виводити результат у консоль).
